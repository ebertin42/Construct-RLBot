# Reward v1 (kill the touch farm) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the flat-touch reward stack (which converged both runs into a non-scoring possession farm at 0.0-0.04 goals/min) with spec §4's shot-making stage: impact-scaled touches, ball-to-goal velocity, offensive potential, goal-dominant with aggression bias — then hot-swap both training runs onto it.

**Architecture:** Extend the existing TOML-driven `reward::RewardConfig`/`compute()` (all new fields serde-defaulted so `reward_v0.toml` keeps byte-identical behavior — regression-tested). All components stateless over the existing `(prev, cur)` GameState pair. New `configs/reward_v1.toml`; `resume_train.py` gains `--reward-config`; both boxes resume checkpoint-exact on the new stack.

**Tech Stack:** existing engine crate (no new deps), serde defaults, existing eval/dashboard goals-metric as the success gauge.

## Global Constraints

- Backward compatibility is a hard gate: `configs/reward_v0.toml` must parse and produce bit-identical rewards to the current code (regression test, exact equality).
- All new components bounded: per-component output in [-1, 1] before weighting (spec §4).
- World-frame geometry: Blue scores into the net at y = +5120, Orange at y = -5120; ball max speed 6000 uu/s; strong-hit velocity delta normalizer 2300 uu/s (0→supersonic ≈ full reward, Nexto convention).
- Goal asymmetry: scorer's team gets `+goal`; conceding team gets `-goal * (1 - aggression_bias)`; `aggression_bias` default 0.0 (= current symmetric behavior), v1 config uses 0.2 (spec §4 stage 3).
- Touch detection reuses the existing verified contract (`ball_hit_info.is_valid` + tick window `(prev.tick_count, cur.tick_count]`) — do not reinvent it.
- Weights live in TOML only; no magic numbers in Rust beyond the physics constants above.
- Existing suites stay green: `cargo test`, `pytest tests/python` (deselect the port-bound render test while the viewer runs).
- Training processes run from this checkout and on the remote — builds are safe; never kill python processes; never modify `checkpoints*/` contents.

```rust
    // Reference copy of the v0 formula, frozen for the regression test.
    fn v0_reference(prev: &GameState, cur: &GameState, car_idx: usize, cfg: &RewardConfig) -> f32 {
        let me = &cur.cars[car_idx];
        let mut r = 0.0f32;
        let hit_now = me.state.ball_hit_info.tick_count_when_hit;
        let hit_before = prev.cars[car_idx].state.ball_hit_info.tick_count_when_hit;
        if me.state.ball_hit_info.is_valid
            && hit_now != hit_before
            && hit_now > prev.tick_count
            && hit_now <= cur.tick_count
        {
            r += cfg.touch;
        }
        let (bp, mp, mv) = (cur.ball.pos, me.state.pos, me.state.vel);
        let d = [bp.x - mp.x, bp.y - mp.y, bp.z - mp.z];
        let dist = (d[0] * d[0] + d[1] * d[1] + d[2] * d[2]).sqrt().max(1e-6);
        let proj = (mv.x * d[0] + mv.y * d[1] + mv.z * d[2]) / dist;
        r + cfg.vel_to_ball * (proj / 2300.0).clamp(0.0, 1.0)
    }
```

```rust
    // --- v1 components (all zero-cost when weights are 0.0) ---
    let opp_goal_y: f32 = if me.team == Team::Blue { 5120.0 } else { -5120.0 };

    if cfg.touch_accel != 0.0 && touched {
        // impact = ball velocity change during the step, normalized by 2300 uu/s
        let dv = [
            cur.ball.vel.x - prev.ball.vel.x,
            cur.ball.vel.y - prev.ball.vel.y,
            cur.ball.vel.z - prev.ball.vel.z,
        ];
        let impact = (dv[0] * dv[0] + dv[1] * dv[1] + dv[2] * dv[2]).sqrt() / 2300.0;
        r += cfg.touch_accel * impact.clamp(0.0, 1.0);
    }

    if cfg.vel_ball_to_goal != 0.0 {
        let g = [0.0 - cur.ball.pos.x, opp_goal_y - cur.ball.pos.y, 0.0];
        let gn = (g[0] * g[0] + g[1] * g[1]).sqrt().max(1e-6);
        let toward = (cur.ball.vel.x * g[0] + cur.ball.vel.y * g[1]) / gn;
        r += cfg.vel_ball_to_goal * (toward / 6000.0).clamp(-1.0, 1.0);
    }

    if cfg.offensive_potential != 0.0 {
        // KRC-2 (Lucy-SKG style geometric mean): sqrt(vel-to-ball+ * ball-goal-alignment+)
        let vtb = (proj / 2300.0).clamp(0.0, 1.0); // reuse the projection computed above
        let bg = [0.0 - cur.ball.pos.x, opp_goal_y - cur.ball.pos.y];
        let bgn = (bg[0] * bg[0] + bg[1] * bg[1]).sqrt().max(1e-6);
        let cb = [cur.ball.pos.x - me.state.pos.x, cur.ball.pos.y - me.state.pos.y];
        let cbn = (cb[0] * cb[0] + cb[1] * cb[1]).sqrt().max(1e-6);
        let align = ((cb[0] * bg[0] + cb[1] * bg[1]) / (cbn * bgn)).clamp(0.0, 1.0);
        r += cfg.offensive_potential * (vtb * align).sqrt();
    }
```
